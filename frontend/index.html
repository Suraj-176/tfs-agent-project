TFS Agent Hub>Quick Start

        

          

            
How the platform works

            
Every agent follows the same 4 stage flow, so anyone can learn it once and use all agents.

            

              
Before you execute

              
    
                
  1. Save TFS Server configuration successfully.
    2. 
                
  2. Save AI Provider configuration successfully.
    3. 
                
  3. Use one small sample first to validate output format.
    4. 
              

            

            

              
Step 1: Select Agent
Pick the agent based on your goal.

              
Step 2: Configure
Fill required inputs shown in the form.

              
Step 3: Execute
Click Execute Agent.

              
Step 4: Review
Check output, logs, and export or upload when available.

            

            

              
For non technical users

              
    
                
  1. Use default values first.
    2. 
                
  2. If you see an error, open the Logs tab and share the message.
    3. 
                
  3. Start with one story or one file before running bulk data.
    4. 
              

            

            

              
For technical users

              
    
                
  1. Validate TFS and provider connections before execution.
    2. 
                
  2. Use fetched story details and prompt overrides only when needed.
    3. 
                
  3. Use output and logs together for root cause analysis.
    4. 
              

            

          


          

            
Setup checklist

            
Complete both setup windows from the top header before using any agent.

            

              
1) TFS Server

              
    
                
  1. Open TFS Server.
    2. 
                
  2. Fill Base URL, Task URL, and Test Plan URL.
    3. 
                
  3. Use either Username/Password or PAT token.
    4. 
                
  4. Click Test Connection, then Save.
    5. 
              

            

            

              
2) AI Provider

              
    
                
  1. Open AI Provider.
    2. 
                
  2. Select provider and add API key.
    3. 
                
  3. For Azure: also fill Endpoint, Deployment, and API Version.
    4. 
                
  4. Click Test, then Save.
    5. 
              

            

            

              
If something fails

              
    
                
  1. Re-open TFS and AI modals and test again.
    2. 
                
  2. Check required fields are not blank.
    3. 
                
  3. Run the agent and inspect the Logs tab for exact error.
    4. 
              

            

          


          

            
Agent 1: Task Creation

            
Creates or updates TFS tasks using Excel, OneDrive, or Google Drive input.

            

              
Inputs

              
    
                
  • Bulk mode: Create or Update
    • 
                
  • Iteration path
    • 
                
  • Input source file or link and sheet name
    • 
              

            

            

              
Step by step

              
    
                
  1. Select Agent 1.
    2. 
                
  2. Choose Create or Update mode.
    3. 
                
  3. Select input method and provide file or link.
    4. 
                
  4. Load and confirm the target sheet.
    5. 
                
  5. Execute and review summary in output and details tabs.
    6. 
              

            

            

              
Output you should see

              
    
                
  • Creation or update summary in Output tab.
    • 
                
  • Item-level details in Details tab.
    • 
              

            

          


          

            
Agent 2: Test Case Generator

            
Generates functional or UI test cases from a story and supports refinement using review chat.

            

              
Inputs

              
    
                
  • User Story ID (recommended) or manual story text
    • 
                
  • SOP text (editable)
    • 
                
  • Mode: Functional, UI, or Both
    • 
                
  • Optional UI screenshots
    • 
              

            

            

              
Step by step

              
    
                
  1. Select Agent 2.
    2. 
                
  2. Enter story ID and click Fetch.
    3. 
                
  3. Set test mode and optional screenshots.
    4. 
                
  4. Execute and review generated table.
    5. 
                
  5. Use analysis chat to ask for gaps or edge cases.
    6. 
                
  6. Upload selected test cases to suite when ready.
    7. 
              

            

            

              
Output you should see

              
    
                
  • Structured test cases with clear steps and expected result.
    • 
                
  • Optional additional cases from review chat.
    • 
              

            

          


          

            
Agent 3: Self-Healing Engine

            
Autonomous system that reviews and repairs agent outputs to ensure zero-error results across the platform.

            

              
Global Capabilities

              
    
                
  • Agent 1 Support: Validates task descriptions and mandatory TFS fields for Tasking Agent.
    • 
                
  • Agent 2 Support: Fixes malformed Markdown tables for perfectly formatted Test Case grids.
    • 
                
  • Agent 4 Support: Sanity-checks reproduction steps and triage fields for Bug & Feature Agent.
    • 
              

            

            

              
Intelligent Repair Process

              
    
                
  1. Autonomous Correction: Instantly detects missing columns or formatting errors.
    2. 
                
  2. Format Enforcement: Enforces professional TFS standards for all work item types.
    3. 
                
  3. Zero-Intervention: Repairs happen silently in the background before you see the result.
    4. 
              

            

          


          

            
Agent 4: Bug and Feature

            
Converts raw issue notes into structured TFS bug or feature work items.

            

              
Inputs

              
    
                
  • Description in chat
    • 
                
  • Optional screenshots
    • 
                
  • Triage fields: area, iteration, severity, priority, tags, assignee
    • 
              

            

            

              
Step by step

              
    
                
  1. Select Agent 4 and choose Bug or Feature.
    2. 
                
  2. Describe the issue in chat.
    3. 
                
  3. Review auto-filled title and description.
    4. 
                
  4. Add or adjust triage fields.
    5. 
                
  5. Execute to create, or use Update Existing mode to modify.
    6. 
              

            

            

              
Output you should see

              
    
                
  • Created or updated work item confirmation.
    • 
                
  • Captured fields visible in output and logs.
    • 
              

            

          


          

            
Agent 5: Dashboard

            
Builds QA metrics dashboard using TFS queries and optional Excel inputs.

            

              
Inputs

              
    
                
  • Query IDs or URLs: Bug, Retesting, Story, Other
    • 
                
  • Optional Excel files for validation or automation
    • 
                
  • Mode: Static or AI
    • 
              

            

            

              
Step by step

              
    
                
  1. Select Agent 5.
    2. 
                
  2. Enter queries and optional files.
    3. 
                
  3. Choose Static or AI mode.
    4. 
                
  4. Execute and open Dashboard tab.
    5. 
                
  5. Download as PDF or image for sharing.
    6. 
              

            

            

              
Output you should see

              
    
                
  • Dashboard charts and counts in Dashboard tab.
    • 
                
  • AI summary narrative when AI mode is selected.
    •
